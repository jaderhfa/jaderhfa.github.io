/* História - como o desenvolvedor modela o criação do jogo tetrinet
cliente e sevidor, para que todos os envolvidos, gerentes ou membros da equipe de desenvolvimento possa entender o jogo tetrinet */
/* Grupo devel-docs */

@startuml

autonumber
actor Cliente
boundary Computador
Cliente -> Computador : Usuário no computador\njogando Tetrinet
Computador -> Tetrinet : O usuário precisa Girar\nPeça no jogo
Computador -> Cliente : Para Girar a peça\nele deve escolher uma das teclas
Cliente  -> Computador : O jogador quer girar a peça a direita entao ira  pressionar \na tecla direcional UP (padrão) e a peça ira rotacionar
Cliente  -> Computador : O jogador quer girar a peça a esquerda entao ira pressionar \na tecla CTRL RIGHT (padrão) e a peça ira rotacionar
Cliente -> Cliente: Usuário escolhe a tecla que deseja\nClica na tecla para movimentar
Computador -> Tetrinet : Computador recebe movimentos\n e envia para Jogo Tetrinet 
Tetrinet -> Tetrinet : Movimento e recebido com sucesso

@enduml
